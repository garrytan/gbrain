# 工作流程

## 標準流程

1. **規劃** — 多步驟任務先呼叫 `todo_write` 列出步驟
2. **載入技能** — 呼叫 `skill_load` 取得領域指引
3. **搜尋記憶體** — 呼叫 `search_memory` 抓取相關背景
4. **執行** — 逐步完成 todo，每步呼叫 `todo_update`
5. **持久化** — 重要步驟後用 `write_memory` 寫入進度
6. **摘要** — 任務完成後在 `mem/summary-<slug>` 寫入摘要
7. **結束** — 純文字回覆使用者，之後不再呼叫工具

## 搜尋策略

```
1. 先用 hybrid 搜尋（lex + vec）
2. 若回傳空且內容剛寫入 → 改用 keyword 模式重試
3. 仍然空 → 內容不存在或索引尚未更新（等 60-90 秒）
```

## 完成條件

收集到足夠資訊並完成所有行動後，用純文字寫出最終答案。
最終回覆後不再呼叫任何工具。
