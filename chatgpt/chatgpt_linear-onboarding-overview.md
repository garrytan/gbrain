## Assistant

**Linear onboarding** refers to a **sequential, step-by-step onboarding flow** where every new user is guided through a **fixed series of screens or actions in order** before they can be considered “set up” or fully introduced to a product. It’s often called **sequential onboarding** in UX design. citeturn0search5

**Key characteristics:**
- **Fixed order:** Users must complete each step in the prescribed sequence before moving on. citeturn0search5  
- **Action-driven:** Typically walks the user through essential setup tasks (e.g., create account, enter data, configure settings) to ensure core requirements are met. citeturn0search5  
- **Low user choice:** Little deviation based on user type or behavior — everyone follows the same path. citeturn0search5  
- **Clear progression:** Often includes progress indicators or checklists to show where the user is in the flow. citeturn0search5

**When it’s used:**  
Products that require certain information or configurations before use (e.g., setting up a workspace, entering required preferences) commonly use linear onboarding because it *ensures completion of necessary tasks*. citeturn0search5

**Trade-offs:**  
- **Simple to build and predictable** for new users. citeturn0search5  
- **Can feel rigid or unnecessary** for experienced users or those with different goals — modern UX often enhances it with contextual or adaptive cues instead. citeturn0search4
