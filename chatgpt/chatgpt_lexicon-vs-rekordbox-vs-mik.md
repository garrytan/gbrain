Below is a comparison of these tools across key preparation aspects, with pros and cons for each:

## Auto Cue Point Generation  
- **Lexicon:** Offers an advanced **Cue Point Generator** that uses machine learning to detect track structure (drops, breakdowns, outro fades) and place cue points automatically【2†L121-L129】【2†L132-L139】. Pros: Highly flexible (user can choose which cue types to generate or even set manual anchor points for the algorithm to use) and batch-processes many tracks at once【17†L531-L540】【17†L543-L551】. Cons: Still not 100% foolproof – certain genres (e.g. reggae or live-band music) confuse the detector, so manual adjustment is sometimes needed【2†L128-L137】【2†L130-L134】. Also, Lexicon’s cue generation is a paid feature (the software requires a subscription), so cost can be a factor for some.  
- **Rekordbox 7:** Historically, Rekordbox did *not* auto-generate multiple cue points; DJs had to set cues manually. Plus, Rekordbox will always set a basic **auto-cue at the first beat** of a track if you enable that option, ensuring every track has a starting cue. Cons: Intelligent Cue Creation depends on having a history of well-set cues (it won’t magically know where a drop is unless your other tracks’ cues give it hints), so it may be less useful for brand-new DJs or when adding vastly different genres. It’s also reportedly locked behind a subscription in Rekordbox 7 (after a trial period)【39†L74-L82】【39†L76-L80】, meaning you might need the paid plan or a hardware unlock to use the advanced auto-cue feature. Cons: Rekordbox sometimes errs on initial analysis, especially with breaks, halftime drum patterns, or songs that have silence before the first downbeat. In short, MIK will get you in the ballpark for BPM and usually matches the other software on simple tempos, but it’s not as *adaptable* as Rekordbox’s dynamic grid or Lexicon’s manual editing in handling complex timing.
