# The product architecture

## 1. Artist Context Graph

This is the foundation. Jovie should build a living profile of the artist from verified and permissioned sources.

### Sources to ingest

| Source type | Examples | Use |
|---|---|---|
| **Music catalog** | Spotify, Apple Music, SoundCloud, Bandcamp, YouTube Music, stems, lyrics, ISRCs | Hook detection, release planning, lyric assets, remix ideas |
| **Owned socials** | TikTok, YouTube, Instagram, X, Threads, Twitch, Discord | Voice, audience, historical winners, content cadence |
| **Video memory** | Vlogs, interviews, podcasts, behind-the-scenes, livestreams | Personal stories, tone, visual identity, “things only this artist would say” |
| **Public context** | Press, interviews, podcasts, live sets, reviews, Wikipedia/Wikidata, MusicBrainz | Enriched memory with provenance |
| **Commercial data** | Shopify, Fourthwall, Patreon, Bandsintown, Songkick, email/SMS, ticketing | Monetization opportunities |
| **Audience data** | Comments, DMs, fan questions, saves, shares, superfan signals | Content prompts, merch ideas, fan segmentation |
| **Creative assets** | Logos, fonts, photos, album art, videos, brand colors | Merch, ads, carousels, visuals |
| **User preferences** | Boundaries, topics, approved claims, humor level, politics, sexuality, style | Prevents bad AI behavior |

### Verification system

For “is this actually David Guetta?” or any public artist, Jovie needs a **confidence-scored identity resolver**:

- Official links from Spotify/Apple/YouTube artist profiles.
- Verified social handles.
- OAuth ownership where possible.
- Public knowledge graph IDs.
- Cross-linked bios and official websites.
- Voice/face similarity only as a supporting signal.
- Manual artist/team confirmation for low-confidence context.
- Provenance on every memory: **owned**, **official**, **public third-party**, **fan-made**, **uncertain**.

This matters because public interviews can be valuable context, but Jovie needs to know whether it is learning from the actual artist, a parody, a fan edit, a different person with the same name, or an unreliable source.

# Highest-value workflows to build

## Priority 1: TikTok / Reels / Shorts Reactivation Agent

This is the immediate wedge for your own use case.

### What it does

Jovie takes a dead or weak social channel and revives it with a daily testing machine.

Inputs:

- Catalog.
- Lyrics.
- Artist persona.
- Existing videos/vlogs.
- Current trend formats.
- Platform analytics.
- Audience comments.
- Approved visual assets.

Outputs:

- 30–90 short-form videos.
- Caption variants.
- Hashtag variants.
- Posting schedule.
- Performance scoring.
- Follow-up videos based on winners.

### Content formats

| Format | Description |
|---|---|
| **Tension loop** | Rope/car style: viewer waits for payoff while learning the hook |
| **Absurd performance** | Still image / uncanny valley / cheap green screen / ridiculous background |
| **Lyric punchline** | One lyric line, one visual joke, one repeatable format |
| **Fake podcast clip** | Artist appears to explain the song in a staged interview format |
| **Vlog memory remix** | Old vlog moment + song hook + text setup |
| **“POV” format** | Song placed into a relatable emotional/social scenario |
| **Duet bait** | Leaves space for fans to react, sing, stitch, or add a verse |
| **Comment reply series** | Turns audience comments into new videos |
| **Behind-the-song microstory** | 15 seconds explaining a line, sample, breakup, joke, or scene |
| **Green-screen pop culture riff** | Artist responds to a public moment using their song as the punchline |

### Key product principle

Do not just repost viral videos with the song underneath. That creates copyright/platform risk and often produces low-trust sludge. Build a **trend recreation engine**:

- Detect the viral format.
- Extract the structure: hook, pacing, tension, caption grammar, payoff.
- Generate an original version using owned/licensed footage or creator-submitted clips.
- Add the artist’s song.
- Publish.
- Measure.

For “viral videos with my song behind them,” the better product is a **UGC / clipping bounty system**: creators submit original videos using the sound; Jovie tracks views, pays winners, and handles disclosure/compliance. Clipping campaigns have become a known social-video marketing tactic, but they are controversial when undisclosed or deceptive. citeturn593981view0 citeturn593981view1

# The key risks to handle in-product

## 1. Copyright / reposting

Build for **recreation, licensing, creator submission, or commentary**, rather than “rip viral video and put my song under it.” YouTube notes that fair use is case-by-case, and TikTok’s copyright policy says copyright protects original works such as videos and music. citeturn958108search26 citeturn958108search7

## 2. Synthetic identity

For virtual artist content, only synthesize the opted-in artist or approved collaborators. Require AI labels where platforms require them. citeturn503513view5 citeturn503513view6

## 3. Fake social proof

Do not fabricate shows, press, streams, crowds, endorsements, or quotes. Stylized content is fine. False commercial proof creates trust and legal risk.
