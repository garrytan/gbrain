## Biggest UI / UX issues

### Navigation & layout
- **Nav overload:** hamburger menu *and* a 5-item tab bar on every screen → redundant and eats vertical space.
- **Excessive whitespace / poor information density:** key content sits far below the fold (especially Analytics + Chat).
- **Inconsistent header behavior:** some screens show extra header elements that crowd/overlap content (Audience screen looks cramped at the top).

### Responsiveness / mobile fit
- **Desktop table shoved into mobile (Audience):** columns truncate, header is cut, and pagination controls (“Rows per page / Previous / Next”) are pointless on mobile with 4 rows.
- **Text truncation bugs:** location shows URL encoding (e.g., `Los%20Angele...`, `Santa%20Monic...`) and some field labels are cut off (“Paste any link (Spotify…”).

### Analytics comprehension
- **Metric inconsistency:** shows **0 total page visits** but **2 unique visitors** (impossible or definition mismatch).
- **Funnel is visually unclear:** “0%” arrows without explaining what the % represents; “PRO VIEW” watermark/overlay reduces legibility.
- **Empty states are dead ends:** “No city data yet / No referrer data yet” without telling the user what to do next.

### Feedback & error handling
- **Generic error in Chat:** “An error occurred. Please try again.” with no retry action, no cause, no preservation of the user’s prompt, no recovery path.

### IA / content mismatches
- **Contacts empty state copy doesn’t match actions:** text mentions *press contacts* but only offers *bookings* and *management* buttons.
- **Integration flow is high-friction + low-trust:** asking users to fetch/paste a Bandsintown API key is cumbersome; the disabled “Save API Key” doesn’t explain what’s missing.

## Recommendations (prioritized)

### P0 — Fix correctness + trust killers
1. **Decode and sanitize displayed data** (e.g., convert `%20` to spaces everywhere; validate location strings).
2. **Reconcile analytics definitions** so counts can’t contradict (or label them clearly, e.g., “identified visitors” vs “page views tracked”).
3. **Replace generic chat errors** with:
   - inline retry button
   - keep the user’s message intact
   - show “temporary issue” + optional error code
   - clear next step (“Try again”, “Check connection”, “Contact support”)

### P1 — Make it actually mobile-first
4. **Replace the Audience table with mobile cards** (stacked fields: Visitor / Type / City / Timestamp / Source). Remove “Rows per page” on mobile.
5. **Fix truncation rules**: allow wrapping for long labels/values; use consistent ellipsis; ensure safe padding.
6. **Reduce header/nav height**: choose **either** bottom tabs **or** hamburger. Prefer:
   - bottom tabs for top 4–5 core sections
   - “More” for everything else
   - remove hamburger unless it opens truly secondary settings

### P2 — Improve activation & “what do I do next?”
7. **Upgrade empty states into guided CTAs**:
   - “No referrer data yet” → “Share your link” + copy button + UTM tips
   - “No city data yet” → “Get your first 10 visits” checklist
8. **Fix Contacts IA**: add the missing “Press contact” CTA or change the copy. Make one primary button style consistent.

### P3 — Reduce friction on integrations
9. **Replace API-key paste with OAuth/connect flow** if possible (“Connect Bandsintown”).
10. If API key must remain:
   - add “Why do we need this?” + security note + revoke instructions
   - add deep link button to the exact Bandsintown settings page
   - show validation errors (“Invalid key”) and success confirmation (“Connected ✓”)

### P4 — Polish & accessibility
11. **Consistent visual hierarchy** (titles, section headers, spacing scale).
12. **Accessibility pass**: tap targets ≥44px, contrast checks, dynamic type support.
