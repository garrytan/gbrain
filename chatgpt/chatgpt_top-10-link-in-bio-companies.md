### A few important caveats (so you don’t over-trust this)

- **Data quality is uneven.**  
  Most of these are private; numbers come from data providers like Tracxn, Growjo, CB Insights, Datanyze, etc., which rely partly on models and self-reported data. They’re good for order-of-magnitude, not precision.

- **“Link in bio” isn’t always the main business.**  
  - For **DistroKid** and **Linkfire**, link-in-bio pages are part of a broader music-distribution / marketing stack.  
  - For **Later**, Linkin.bio is one feature inside a full social-media management suite.  
  Their company revenue is much larger than the slice attributable to bio links alone.

- **User share vs revenue don’t perfectly line up.**  
  One 2024 market report on Instagram link-in-bio tools shows **Linktree as the dominant player by users (≈80% share), with Milkshake, Beacons, Linkin.bio, Bio.site, Lnk.Bio, Instabio, DistroKid, Linkfire, and Carrd rounding out the top 10 by user count**. citeturn0search2turn0search28  
  But Milkshake, Later, DistroKid, etc. appear to monetize more heavily per user than Linktree.

- **What’s missing?**  
  I’ve *intentionally* not ranked giants like **Shopify (Linkpop)**, **Squarespace (Bio Site)**, or **beehiiv’s link-in-bio**, because their core business is e-commerce or email, and bio links are a tiny feature inside a much bigger company—which would drown out the specialist players you probably care about.

## 2. WCAG-ish test you can literally implement

If you want to be rigorous:

- For **non-text UI icons**, WCAG’s non-text contrast success criterion is **3:1** for the icon vs its immediate background.

So you can write a test along these lines (pseudo-Jest):

```ts
// 1. Relative luminance for sRGB
function relativeLuminance(hex: string): number {
  const [r, g, b] = hexToRgb(hex).map(c => {
    const v = c / 255;
    return v <= 0.03928 ? v / 12.92 : Math.pow((v + 0.055) / 1.055, 2.4);
  });
  return 0.2126 * r + 0.7152 * g + 0.0722 * b;
}

// 2. WCAG contrast ratio
function contrastRatio(a: string, b: string): number {
  const L1 = relativeLuminance(a);
  const L2 = relativeLuminance(b);
  const [lighter, darker] = L1 > L2 ? [L1, L2] : [L2, L1];
  return (lighter + 0.05) / (darker + 0.05);
}

const ICON_MIN_CONTRAST = 3; // WCAG non-text target

test('social icons meet contrast requirement', () => {
  const bg = '#050509'; // your dark bg token

  const iconColors = {
    instagram: '#E4405F',
    youtube: '#FF0000',
    x: '#000000',
    tiktok: '#000000',
  };

  for (const [name, fg] of Object.entries(iconColors)) {
    const ratio = contrastRatio(fg, bg);
    expect(ratio).toBeGreaterThanOrEqual(ICON_MIN_CONTRAST);
  }
});
```

Then you iterate on your design tokens until that passes.  
For X and TikTok you’ll quickly see `contrastRatio(black, dark-bg) < 3`, which tells you “this must be white (or near-white).”
