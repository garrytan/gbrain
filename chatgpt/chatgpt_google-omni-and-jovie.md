## Important legal / operational caveats

For scrappy testing, use lightweight reference capture: phone photos, short clips, no blocking sidewalks, no drones, no staged commercial crew, no identifiable bystanders.

Once this is commercial Jovie content, location capture and filming in LA can trigger permit and insurance issues. FilmLA says City of Los Angeles rules require a temporary filming permit for on-location filming of a commercial nature on public and private property within city jurisdiction, and FilmLA also says personal/non-commercial filming generally does not require a permit. citeturn783732search1turn783732search3

# MVP shipping sequence

## 0. Define the core data model first

Even before identity generation is good, make the product architecture identity-first.

### Core objects

| Object | Meaning |
|---|---|
| **Identity** | Real artist, virtual artist, influencer, character, founder, model |
| **Song** | Track, hook, stems, lyrics, genre, mood, BPM |
| **Scene** | Generic scene, LA location, performance setup, lifestyle moment |
| **Render** | The actual TikTok/Reel/Short output |
| **Memory** | What worked, what the artist liked, what performed |

The primitive is:

```text
Identity + Song + Scene = Render
```

Day one identity schema:

```text
Identity
- owner_user_id
- identity_type: real_person | virtual_character
- display_name
- visual_references[]
- style_card
- music_assets[]
- voice_profile: null initially
- consent_status
- allowed_output_types
- prohibited_output_types
- generated_assets[]
- performance_memory[]
```

This lets you start with generic videos while already laying the foundation for locked identities.

# 7. V5: voice identity

Voice should come after visual identity and approval workflow.

### Start with low-risk voice use cases

| Use case | Example |
|---|---|
| Spoken intro | “This one’s called…” |
| Artist tag | “Jovie made this for my track.” |
| Skit voiceover | Artist narrates a scene |
| Promo ad read | “Presave my new single.” |
| Character dialogue | Virtual identity speaks |

### Defer harder voice use cases

| Harder use case | Why |
|---|---|
| Singing voice clone | Higher rights and quality burden |
| Duets with other artists | Likeness/voice permission issues |
| Celebrity-style voices | High legal and platform risk |
| Public figure voice | High impersonation risk |

For California specifically, AB 2602 defines “digital replica” as a highly realistic computer-generated representation identifiable as an individual’s voice or visual likeness, and it addresses contract enforceability for using a person’s digital replica in place of work they would otherwise perform. citeturn664846view0

For product safety, require:

```text
- Liveness check
- Consent phrase
- Voice sample ownership confirmation
- Allowed use types
- Revocation controls
- Audit log
- Output approval before commercial use
```

# Important platform constraints

TikTok says realistic AI-generated images, audio, and video must be labeled, including cases where a real person is portrayed doing or saying something they did not do or say. TikTok also says it prohibits using the likeness of adult private figures without permission. citeturn803774view5

If Jovie posts promote Jovie, an artist’s business, or a brand, TikTok says commercial content needs the content disclosure setting turned on. citeturn803774view6

So the product needs built-in disclosure handling:

```text
- AI-generated label reminder
- Commercial-content disclosure reminder
- Consent log
- Identity owner approval
- No unauthorized third-party likenesses
- No minors
- No public-figure impersonation
```

Build Jovie around **Scene Cards** now.

```text
SceneCard
- name: Sunset Strip Night Walk
- source_type: generic | user_uploaded | owned_capture | street_view_grounded | model_generated
- location_label
- mood
- camera_style
- lighting
- wardrobe_defaults
- prompt_template
- negative_prompt
- legal_notes
- example_outputs
```

This lets you use different backends:

```text
Google Flow / Genie manually
Vercel AI Gateway
Veo
Omni
Higgsfield
Kling
Seedance
future Maps Imagery Grounding API
owned LA reference footage
```

The user never cares which model made it. They care that it looks like a postable TikTok.

## Legal/product caution

Do **not** build by scraping or screenshotting Google Street View. Google’s geo guidelines say Street View imagery should not be screenshotted, removed from embedded sources, downloaded, or used separately from Google services. citeturn809150search0

So the product path is:

```text
Use Google’s official grounding tools where available
Use your own captured reference footage where needed
Use generated approximations for MVP testing
Avoid claiming exact real-location reproduction unless rights/source permit it
```

## What this means for the LA mapping idea

Manual LA mapping is still useful, but the purpose changes.

### Old idea

```text
We map LA because nobody else has the location data.
```

### Better idea

```text
We curate LA because artists need culturally correct scenes.
```

Google can know what Sunset Strip looks like. Jovie should know how a rising rapper, DJ, indie artist, model, or founder should appear there in a TikTok.

That is taste, format, identity, and workflow.
