## Assistant

This GitHub Actions workflow is for building and uploading a Chrome extension. The workflow is triggered when a new tag is pushed with a version number in the format of `v*.*.*`.

The workflow consists of two jobs. The first job, `build-chrome-extension`, checks out the code, installs the dependencies, builds the extension, and creates a zipped artifact of the extension. The artifact is uploaded using the `actions/upload-artifact` action.

The second job, `upload-extension`, runs on the `ubuntu-latest` operating system and is dependent on the `build-chrome-extension` job. This job downloads the zipped artifact, installs the `chrome-webstore-upload-cli` tool, and uploads the extension to the Chrome Web Store using the `chrome-webstore-upload` CLI command. The extension ID is specified using the `EXTENSION_ID` environment variable.

The Google API keys required for the upload are stored as secrets in the repository settings, under the names `CI_GOOGLE_CLIENT_ID`, `CI_GOOGLE_CLIENT_SECRET`, and `CI_GOOGLE_REFRESH_TOKEN`. These secrets are used in the `chrome-webstore-upload` command to authenticate the upload.

Overall, this workflow provides a streamlined process for building and uploading Chrome extensions.

## Assistant

Yes, you can use GitHub Actions to update the version number in your manifest file to match the git tag. Here's one way to do it:

1. Add the `actions/checkout` action to your workflow to checkout the code into the workflow's working directory.

2. Use the `actions/setup-node` action to set up Node.js in the workflow.

3. Install the `npm` package `json` using `npm install -g json` to be able to manipulate the JSON file.

4. Use the `run` step to update the `manifest.json` file using the `json` command-line tool. This can be done by setting the version key in the `manifest.json` file to the git tag. You can get the git tag from the `GITHUB_REF` environment variable.

Here's an example of what the step might look like:

```
- name: Update manifest file
  run: |
    json -I -f manifest.json -e "this.version='${{ env.GITHUB_REF/refs/tags/v/ }}'"
```

This command updates the `version` key in the `manifest.json` file to match the git tag. Note that we're using `${{ env.GITHUB_REF/refs/tags/v/ }}` to remove the leading `v` from the git tag (assuming that your tags start with a `v`). If your tags don't start with a `v`, you can omit the `/refs/tags/v/` part.

Make sure to add this step to the `build-chrome-extension` job before the `zip` command so that the updated manifest file is included in the zipped artifact.

```
name: Update Chrome Extension

      - v*.*.*

          # Build the extension

      # all of our environments (dev, staging, prod) have a consistent ID
      # we can reference. Otherwise the extension ID is autogenerated.

    steps:
      - uses: actions/setup-node@v2-beta
```

The `-I` option tells the `json` tool to modify the file in place, and the `-f` option specifies the name of the file to modify.
