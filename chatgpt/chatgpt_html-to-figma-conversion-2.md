You **can convert HTML (code or live websites) into editable Figma designs** — there’s no manual built-in feature in Figma, but there are reliable tools and plugins that do exactly that. citeturn0search2turn0search3turn0search14

### 👍 Best Tools to Convert HTML → Figma

**1. html.to.design (Figma Plugin + Chrome Extension)**  
- Lets you import an existing webpage or HTML/CSS code into Figma as **fully editable layers**. citeturn0search2  
- Works by either:  
  - **Pasting a URL** directly in the Figma plugin, or  
  - Using a **Chrome extension** to capture a page and send it straight into Figma. citeturn0search3  
- **New feature:** paste raw HTML/CSS directly in the plugin’s **HTML tab** and convert it to editable Figma elements. citeturn0search14

**Workflow example:**  
1. Install the **html.to.design** Figma plugin.  
2. (Optionally) install its Chrome extension.  
3. Either paste the website URL in the plugin or navigate to the site in Chrome and use the extension.  
4. Import → get editable frames, auto layout, components inside Figma. citeturn0search3

**2. Codia HTML to Design / Web2Figma Tools**  
- Alternatives that also convert websites/code into editable Figma designs using AI. citeturn0search5  
- Usually works via a Chrome extension or direct import into Figma.

### 🧠 Tips & Notes
- These tools aren’t perfect — results vary with layout complexity and dynamic elements (JS animations etc.).  
- Accuracy is generally high (80–90% realistic structure). citeturn0search10  
- Imports usually maintain visual hierarchy and styles, letting you tweak them as native Figma components.
