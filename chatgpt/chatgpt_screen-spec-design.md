## Assistant

Alright Mr. White, here’s a concrete spec that merges:

- **Blue chart style** (your favorite)
- **MacroFactor-level functionality**
- **Apple Health cleanliness**
- **Your current purple identity** (without the heavy boxes)

---

## 1. Overall screen structure

Top → bottom stack with generous 16pt side margins, 8pt vertical rhythm.

1. **Nav bar**
   - Left: chevron/back.
   - Center: metric name (e.g., *Body Fat %*).
   - Right: “+” add button.
   - All icons SF Symbols, 22pt, weight semibold.

2. **Headline metric block**
   - Large value: `10.3` in **SF Pro Display, 60–64pt, regular**.
   - Unit: `%` in 20pt, medium, baseline-aligned.
   - Subline: date (`Nov 11, 2025`) in 15pt, secondary color.

3. **Quick stats row (MacroFactor-style)**
   - Row of 2–3 compact stats:
     - **Average** (e.g., `30d avg 10.1%`)
     - **Change** (e.g., `Δ −0.6% since 30d ago`)
     - Optional **Range** (`9.6–10.7%`)
   - Each stat: label 12pt, value 15pt, slight letterspacing.
   - Use subtle separators (·) instead of lines to keep it Apple-clean.

4. **Time range selector**
   - Single horizontal pill control: `1W  1M  3M  6M  1Y  All`
   - Height 32pt, pills 64–72pt wide, 8pt gap.
   - Active pill: filled with **accent**; inactive: hairline border.

5. **Trend card (chart)**
   - Rounded rectangle, **20pt radius**, full-width minus 16pt margin.
   - Title left (`Trend`), small toggle on right (`Raw / Smoothed` chip).

6. **History list**
   - Apple Health style: flat cells, no heavy card box, full-width tap targets.

---

## 2. Color & type system (dark mode)

### Backgrounds
- **App background:** `#000000`
- **Card background:** `#080A0D`
- **Secondary surfaces (history rows):** `#050507`

### Primary accent
- Keep your **purple/pink** for *UI*; use **blue for data ink**.

Suggested:
- **Accent (UI, chips, icons):** `#D32AFF`
- **Chart line blue:** `#4FD1FF`
- **Chart fill gradient:**
  - Top: `rgba(79, 209, 255, 0.25)`
  - Bottom: `rgba(79, 209, 255, 0.00)`

### Text
- Primary: `#FFFFFF`
- Secondary: `rgba(255,255,255,0.65)`
- Tertiary / captions: `rgba(255,255,255,0.40)`
- Success/failure deltas (for change stat):  
  - Down in body fat: `#4CD964`  
  - Up in body fat: `#FF453A`

Use **SF Pro Display** for big numbers, **SF Pro Text** for everything else.

---

## 3. Trend chart – visual details

**Container**
- Height ~260pt.
- Padding: 16pt sides, 16pt top, 12pt bottom.
- Corner radius: 20pt.
- Inner shadow very subtle (`rgba(0,0,0,0.45)`) to “sink” it.

**Grid**
- Horizontal grid lines only (no vertical) at ~4–5 levels.
- Color: `rgba(255,255,255,0.08)`, 1pt.
- Y-axis labels: 11pt, tertiary color; x-axis labels only at extremes + mid.

**Line**
- Single smooth line, 2pt stroke, **no dots by default**.
- When a point is selected: 6pt filled circle with white border.
- Line joins rounded, no sharp angles.

**Fill**
- Under-line gradient (blue → transparent) starting at ~70% of card height.

**Interaction**
- Long press or drag:
  - Vertical dashed guideline `rgba(255,255,255,0.4)`, 1pt.
  - Floating pill above the line: white pill, text `9.6%` + date in 11pt.
  - Card header value (top of screen) syncs to the selected point.

**Trend vs raw (MacroFactor behavior)**
- Toggle chips in top-right of card:
  - `Raw` / `Trend`
  - Active chip filled with accent, inactive outlined.
- In **Trend** mode: show smoothed curve (e.g., 7-day moving average).
- In **Raw** mode: show tiny dots for each measurement plus line; no fill.

---

## 4. Time range selector behavior

- When user taps `1W`, `1M`, etc.:
  - Chart animates with a quick easing (`0.25s`) to new range.
  - Stats row recalculates using the visible range:
    - `Average` = average over selected range.
    - `Δ` = change from first visible point to last.

- Scrollable on small devices if needed; keep active pill centered when possible.

---

## 5. History list

Make it feel like Apple Health but tie into your palette.

**Row layout**
- Left column:
  - Date (`Nov 13, 2025`) 15pt, primary.
  - Source (`Apple Health`, `Manual`, `Scale`) 12pt, tertiary.
- Right column:
  - Value (`163 lbs`) in 16pt, medium, right-aligned.
  - Secondary value (`10.7 %`) in 13pt, secondary, right-aligned.

**Styling**
- Row height: 56–60pt.
- No card boxes; just separators:
  - One-pixel divider `rgba(255,255,255,0.08)` under each row except last.
- Selected row: very soft highlight:
  - `backgroundColor: rgba(255,255,255,0.06)`
  - Left-edge accent bar 2pt in purple.

---

## 6. Micro-interactions / polish

- **Haptic** on chart scrub start + on timeframe change.
- **Subtle bounce** when hitting chart edges while scrubbing.
- **Crossfade** when metric changes (Weight → Body Fat %, etc.) rather than hard switch.
- Maintain **one corner radius token** (e.g., 20 for main cards, 12 for pills, 8 for chips) and stick to it everywhere.

---

## 7. TL;DR design recipe

- Use **blue gradient chart** from your favorite variant, on a **muted black/ink card**.
- Layer **MacroFactor-style stats + trend toggle** above that chart.
- Wrap everything in **Apple-simple layout**: clean typography, few borders, lots of breathing room.
- Keep **purple** as brand accent for icons/chips; keep **all data ink in blue** for consistency.
