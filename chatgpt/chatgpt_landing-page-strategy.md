## Assistant

**Mr. White, here is the direct answer: lead with the operating system. Show the phone as proof.**

You are selling two different narratives:

1. **“Link in bio” tool → phone-first visual**
2. **“Operating system for your music career” → desktop dashboard visual**

If you position as an OS for serious artists, the hero must communicate power, scale, and control. That is desktop.

- Center: Releases table (multiple upcoming + past releases)
- Left sidebar: Navigation (Releases, Capture, Analytics, Profile, etc.)
- Right slide-over panel: Single release expanded  
  - DSP links (Spotify, Apple Music, SoundCloud, etc.)

  - Scheduled date
  - Status badge (Scheduled / Live)

**Why this works:**
- Signals “professional tool”
- Signals scale (many releases at once)
- Shows automation visually (scheduled + notifications)
- Creates FOMO for high-output artists (DJs, remixers, producers)

This is your **Linear-style credibility moment**.  
It communicates: “This is infrastructure.”

## 2. Below the Fold: Show the Modes (Phone Mockups as Proof)

This section answers the subconscious question:

> “What does the fan actually see?”

Show 3–4 phone screens:

- **Listen Mode** – Smart links across DSPs  
- **Subscribe Mode** – Email capture  
- **Tour Mode** – Tour dates  
- Optional: **Release Notification Mode**

Each screen labeled clearly:
- “Listen”
- “Subscribe”
- “Tour”

This communicates:
- Yes, it’s a link in bio.
- Yes, it captures emails.
- Yes, it handles tour.
- It all lives in one page.

You do not need the phone in the hero if this section exists clearly.

---

## 3. Positioning Logic

If headline =  
**“The Operating System for Your Music Career”**

→ Hero = Desktop OS  
→ Phone = supporting proof

If headline =  
**“The Smartest Link in Bio for Artists”**

→ Hero = Phone  
→ Desktop = supporting power feature

You cannot visually lead with both at the same time without diluting clarity.

---

## 4. Recommended Structure (High-Conviction Version)

1. **Hero**
   - Big desktop dashboard mockup
   - Headline: OS positioning
   - Subheadline: automation + scheduling + email + notifications
   - CTA: Claim handle

2. **Section 2**
   - “Everything lives on one page”
   - 3–4 phone mockups showing modes

3. **Section 3**
   - Release automation explanation
   - Highlight: “Schedule once. Fans notified automatically.”

4. Optional
   - Small floating phone graphic near footer if you want reinforcement

---

## 5. What Actually Drives FOMO

DJs and prolific artists respond to:
- Volume control (many releases visible)
- Automation (scheduled + notifications)
- Professional UI (dark, clean, sharp)

The full releases table mockup hits that harder than a phone ever will.

The phone communicates usability.  
The desktop communicates power.

You want power first.
