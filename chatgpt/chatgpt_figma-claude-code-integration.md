Here’s what “**Figma + Claude Code**” means and how to work with it — including relevant code-oriented workflows.

### 🧠 What the integration does
- **Claude Code to Figma:** This new feature lets you **capture a running UI (from a browser preview)** generated by Claude Code and **import it into Figma as fully editable design frames** (not just flat screenshots). citeturn0search0turn0search6  
- It uses Figma’s **Model Context Protocol (MCP) server** as the bridge between Claude Code and Figma’s design environment. citeturn0search3

### ⚙️ Typical workflow (developer + design)
1. **Set up the MCP connection**  
   You add Figma as an MCP target inside Claude Code so it can communicate with your Figma workspace. citeturn0search3

   Example CLI command:
   ```sh
   claude mcp add --transport http figma <https://mcp.figma.com/mcp>
   ```

2. **Authenticate Figma**  
   In Claude Code, run the `/mcp` command and authenticate with Figma’s OAuth flow so Claude has permission to send design data.

3. **Build UI in Claude Code**  
   Write code or prompts in Claude Code that generate a running UI (e.g., a web component).

4. **Send to Figma**  
   Once your UI is rendered in the browser, type a directive like:

   ```
   Send this to Figma
   ```

   Claude captures the live UI and places it on your Figma canvas as editable elements. citeturn0search3

### 🧑‍💻 Example prompt pattern
Inside Claude Code you might use something like:

```txt
Build a responsive signup form using Tailwind CSS.
Preview it in the browser and then send it to Figma.
```

Then after preview:

```txt
Send this to Figma
```

This asks Claude to invoke the MCP capture and add the UI as layers to your Figma file. citeturn0search3

### 🛠 How it works under the hood
- **MCP Server:** Figma’s MCP acts like an API layer that Claude Code connects to; it lets the AI read design context and generate Figma file content. citeturn0search3  
- **Editable results:** The imported frames in Figma contain editable text, components, and layout, so designers don’t treat it as a static screenshot. citeturn0search6

### 💡 Key limitations & notes
- There’s **a roundtrip cost**: code → Figma → design → back to code, and fidelity isn’t perfect (logic/handlers don’t move back automatically). citeturn0search3  
- This is different from generating designs *in* Figma with an AI prompt; instead it captures *working UIs* from Claude Code. citeturn0search6
