```text
Design a desktop-first, high-converting landing page for Jovie.

CONTEXT
Jovie is the release system for independent artists. It helps independent artists and lean teams plan every release, manage launch tasks, create artist profiles that convert, run pre-save-to-live release destinations, notify fans automatically, and turn every drop into a repeatable growth system.

Use the current product truth from jov.ie as raw material, but redesign the marketing site from scratch for maximum clarity, taste, and conversion.

BUSINESS OBJECTIVE
Maximize free-start signups from cold traffic.

In the first screen, make it obvious in under 3 seconds:
1. what Jovie is
2. who it is for
3. what concrete outcome it creates

USER FEELING
Immediate clarity, trust, momentum, polish, and product credibility.
The visitor should think: “I understand what this does, it looks real, and I want to try it.”

CONVERSION RULES
- Follow YC-style landing page logic: one dominant CTA, minimal competing actions, plain language, fast path to the product aha moment, low-friction signup cues
- Lead with explicit value, not clever slogans
- The current line “Drop more music. Crush every release.” can appear as supporting copy, but the main headline must be more direct
- No jargon
- No vague buzzwords
- No equal-weight primary and secondary buttons
- No carousels
- No animated slides
- No long paragraphs
- No fake testimonials
- No invented metrics
- No pricing table unless pricing is known

BRAND + VISUAL DIRECTION
Create a premium, calm, modern, product-first SaaS landing page with the level of polish and restraint associated with Apple, Frame.io, and Linear. Do not copy their layouts literally. Borrow the feel: precision, confidence, whitespace, and material quality.

Color system:
- Use a monochrome foundation, or a restrained multi-accent system like Apple
- Never use one single accent color across the whole page
- Base palette: black, white, graphite, silver, warm gray
- Accent palette: restrained cobalt, violet, emerald, and warm coral/orange
- Use accents intentionally by module and inside product UI states
- The overall feel should still be premium and controlled, not loud

Typography + layout:
- Bold, crisp, modern typography
- Large negative space
- Tight hierarchy
- 12-column grid
- Hairline borders
- Soft shadows
- Subtle gradients only where useful
- Clean sticky header
- Elegant spacing rhythm
- Quiet depth, not flashy effects

IMPORTANT VISUAL RULES
This must feel like a real shipped product marketing page, not a concept shot.
- No stock photography
- No musicians on stage
- No giant fake laptop frames
- No heavy browser chrome
- No generic abstract SaaS blobs
- Product visuals are the star
- Mockups must look like the real product with minimal chrome and realistic content
- Use album art, release metadata, artist profile cards, links, tasks, notifications, and launch states as tasteful visual material
- High-end mockups should feel almost photographic in polish but clearly be UI
- Device bezels should be thin and understated
- Browser frames should be minimal or nearly invisible

COPY DIRECTION
Use plain, direct, high-clarity copy.

Headline territory:
- “The release system for independent artists.”
- “Run every release from one system.”
- “Plan the drop. Capture the fan. Launch from one command center.”

Subhead territory:
- “Jovie helps independent artists plan launches, capture fans, and turn every release into a repeatable growth system.”
- “From release plans to smart links to fan capture, Jovie gives every drop a system.”

CTA direction:
- Primary CTA: Start free
- Secondary CTA: See demo
- The primary CTA should dominate visually
- The secondary CTA should feel lighter and quieter
- Include understated microcopy: No credit card required

PAGE STRUCTURE

1. HEADER
- Logo left
- 3–4 nav items max
- One primary CTA on the right
- Minimal, premium, sticky

2. HERO ABOVE THE FOLD
- Strong direct headline
- One concise subhead
- One dominant CTA
- One quieter secondary action
- Lightweight proof row beneath CTA
- The hero visual must immediately show the product

Hero visual composition:
Create a premium composite of real-looking Jovie product surfaces:
- Desktop command center with release plan, task states, metadata checks, launch prep, fan email queue
- Floating mobile artist profile that can switch between growth, tickets, tips, and streams
- Smart link / release page with album art and streaming destinations
- Show realistic details like “Generate Release Plan,” task statuses, pre-save countdown, live release states, and fan notifications

The hero should make the product obvious instantly.

3. SOCIAL PROOF / TRUST BAND
- Elegant, low-noise logo strip
- Use current proof like The Orchard, AWAL, Universal Music Group, Armada Music if appropriate
- Keep this subtle and premium

4. OUTCOME SECTION
Create 3 strong customer-outcome blocks.
Each block should start with the outcome, then show the product proof.

Suggested outcomes:
- Never start from zero on launch day
- Capture the fan first
- Share every release from one destination before and after launch

5. PRODUCT DEEP DIVE
Section title direction:
“A command center for your career.”

Show high-end product visuals for:
- Generate a release plan from a brief
- Tasks track themselves
- Metadata verified
- Launch prep organized
- Editorial pitch tasks surfaced
- Fans notified automatically in their timezone

Use realistic UI states, not placeholders.

6. BEFORE / AFTER RELEASE DESTINATION STORY
Create one of the strongest visual moments on the page:
- Same release destination before launch as a pre-save page
- Same surface after launch as a live smart link
- Show the transformation clearly and beautifully
- This should feel inevitable and useful, not gimmicky

7. HOW IT WORKS
4 concise steps with strong hierarchy and supporting UI:
- Brief loaded
- Release plan generated
- Launch tasks managed
- Fans and links update automatically

Keep it visual, simple, and believable.

8. COMPOUNDING SECTION
Express the idea that with Jovie, each release compounds instead of flatlining.
Do this with a premium, understated visualization integrated into the product story.
Avoid cheesy charts. Make it feel like part of the system.

9. FINAL CTA
Repeat the main promise in a sharper, simpler form.
Use one dominant CTA only.
Make the close frictionless and obvious.

STYLE GUARDRAILS
- Real product > abstract design
- Clarity > cleverness
- Outcomes > features
- Premium restraint > startup noise
- Minimal chrome > framed mockups
- Direct copy > slogans
- Trust > hype

OUTPUT
Generate a complete, high-fidelity, production-ready-looking landing page for the web.
Desktop-first, with clear responsive logic for mobile.
It should feel like a site that could ship today.
Prioritize conversion clarity, taste, and product credibility.
The first screen must make the product obvious in under 3 seconds.
The finished page should feel like premium software for serious independent artists and their teams.
```
