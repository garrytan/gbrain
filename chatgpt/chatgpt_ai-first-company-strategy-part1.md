# The strategic framing

Jovie should become the AI agent that helps artists run the business side of being an artist.

That includes:

- release planning
- content creation
- fan alerts
- merch drops
- relationship warming
- collaboration outreach
- sync/licensing outreach
- social posting
- email/DM drafting
- lightweight CRM
- sales/fundraising-style pipeline work
- KPI feedback loops

This is the same work founders do:

- fundraising = investor relationship management
- sales = customer relationship management
- social = audience development
- product launches = release campaigns
- retention = fan engagement
- partnerships = collabs/sync/label/DSP relationships

So the shared abstraction is:

> **Jovie helps a creative person turn assets, relationships, and attention into revenue.**

For you, the asset is Jovie + your music credibility.  
For artists, the asset is music + audience + catalog.

Same engine. Different templates.

# The operating principle

## Dogfood Jovie as Tim White, artist-founder

You should use Jovie for:

1. **Warming relationships**
2. **Fundraising**
3. **Founder-led social**
4. **Sales/outreach**
5. **Music/content promotion**
6. **Merch experiments**
7. **Release campaign automation**

But early on, the UI does not need to be fully productized.

The mistake would be waiting until the Jovie chat UI supports everything perfectly.

The right move is:

> Build the reusable skill files now. Run them wherever fastest. Productize the winners.

So yes, use Claude Code, Hermes, markdown files, Gmail drafts, Typefully MCP, etc. But treat those as **runners**, not the actual product brain.

The durable asset is the skill/eval/memory system.

This is the highest-leverage first skill because it maps cleanly to both founders and artists.

## Build this first: `relationship_warmer`

### Inputs

- Person
- Relationship history
- Last interaction
- Current goal
- Tone preference

### Output

Most first messages should **not sell**.

They should reopen the loop.

- “Yo man, random but I was just thinking about you. What have you been up to lately?”
- “Been a minute. How’s everything on your side?”
- “Saw your name pop up and realized we haven’t caught up in forever. What’s new?”

Then if they ask what you’re doing, Jovie routes into:

- founder update
- fundraise angle
- music angle
- customer discovery angle
- intro ask

This is the exact same pattern artists need for collaborators, fans, DJs, playlist people, promoters, photographers, videographers, producers, and managers.

# Fifth priority: evals before automation

Before you let Jovie send anything automatically, build draft evals.

## Minimum eval gates

Every outbound message should be scored on:

| Eval | Question |
|---|---|
| Voice match | Does this sound like Tim/the artist? |
| Specificity | Is it grounded in real context? |
| Cringe risk | Does it sound fake, needy, spammy, or over-AI’d? |
| Conversion intent | Is there a clear next step? |
| Relationship fit | Is the ask appropriate for this person? |
| Compliance/safety | Is it okay to send? |
| Edit distance | How much did Tim change before sending? |
| Outcome | Did they reply/book/click/buy? |

Do this manually at first.

Simple flow:

1. Jovie drafts.
2. You edit.
3. Jovie stores original + edited version.
4. Jovie labels why the edit happened.
5. Future drafts improve.

That gives you compounding data.

- new release alert

Examples:

- welcome new fan

## Level 5: Autonomous relationship manager

Jovie manages fan engagement under artist-approved rules.

Rules:

```txt
Never mention personal details unless the fan gave them directly.
Never fake intimacy.
Never pretend the artist personally saw something unless true.
Never over-message.
Always respect preferences.
Escalate emotional, angry, romantic, legal, or high-value messages to artist review.
```
