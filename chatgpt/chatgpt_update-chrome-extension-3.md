name: Update Chrome Extension

on:
  push:
    tags:
      - v*.*.*

jobs:
  build-chrome-extension:
    name: Build Chrome extension artifact
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v2

      - name: Install dependencies
        run: npm install

      - name: Update manifest file
        run: json -I -f manifest.json -e "this.version='${{ env.GITHUB_REF/refs/tags/v/ }}'"

      - name: Build
        run: |-
          npm run prod
         
          # we archive the dist folder and include SHA commit as the last step
          zip -r chrome-extension-${{ github.event.pull_request.head.sha }}.zip dist


      - name: Archive chrome-extension artifact
        uses: actions/upload-artifact@v3
        with:
          name: chrome-extension-${{ github.sha }}
          path: chrome-extension-${{ github.event.pull_request.head.sha }}.zip


           # api usage reference:
  # * <https://developer.chrome.com/docs/webstore/using_webstore_api/>
  # * <https://github.com/fregante/chrome-webstore-upload/blob/main/How%20to%20generate%20Google%20API%20keys.md>
  upload-extension:
    name: Upload extension
    runs-on: ubuntu-latest
    needs: build-chrome-extension
    env:
      # you can optionally specify extension ID here, we do this so that
      # all of our environments (dev, staging, prod) have a consistent ID
      # we can reference. Otherwise the extension ID is autogenerated.
      EXTENSION_ID: daeopelnocmnhbkjbjmhckgjgheioafj

    steps:
      - uses: actions/setup-node@v2-beta
        with:
          node-version: "16.10"

      - name: Download bundle artifact
        uses: actions/download-artifact@v3
        with:
          name: chrome-extension-${{ github.sha }}

      - name: Install webstore cli
        run: |-
          npm install -g chrome-webstore-upload-cli

      - name: Upload step
        run: chrome-webstore-upload upload --client-id ${{ secrets.CI_GOOGLE_CLIENT_ID }} --client-secret ${{ secrets.CI_GOOGLE_CLIENT_SECRET }} --refresh-token ${{ secrets.CI_GOOGLE_REFRESH_TOKEN }} --source chrome-extension-${{ github.event.pull_request.head.sha }}.zip --extension-id ${{ env.EXTENSION_ID }}

## Assistant

This GitHub Actions workflow is for building and uploading a Chrome Extension. Here's a summary of what each step does:

1. The `Checkout` step checks out the code from the repository.
2. The `Install dependencies` step installs any required dependencies using npm.
3. The `Update manifest file` step updates the version number in the `manifest.json` file to match the tag associated with the commit.
4. The `Build` step builds the extension by running the `npm run prod` command and then archives the `dist` folder as a zip file.
5. The `Archive chrome-extension artifact` step uploads the zip file as a build artifact.
6. The `Download bundle artifact` step downloads the build artifact from the previous step.
7. The `Install webstore cli` step installs the `chrome-webstore-upload-cli` package, which is used to upload the extension to the Chrome Web Store.
8. The `Upload step` step uploads the extension using the `chrome-webstore-upload` command-line tool, which requires a Google API client ID, client secret, and refresh token stored as GitHub secrets.

Note that this workflow assumes that the extension ID is specified in the environment variable `EXTENSION_ID`. If it's not, the extension ID will be autogenerated. Also, the workflow uses the `chrome-webstore-upload-cli` tool to upload the extension, which requires you to have generated the necessary Google API keys beforehand.
