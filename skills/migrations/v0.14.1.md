# v0.14.1 Migration: Canonical Target Proposals

This release completes the canonical target proposal workflow for Memory Inbox
candidates that do not yet have a safe compiled-memory page target. It also
hardens the release path after the v0.14.0 tag workflow failed on macOS before
publishing release assets.

## Agent Instructions

When presenting this update, say something like:

"v0.14.1 gives targetless Memory Inbox candidates a proposal-first review path.
I should refresh setup-agent so Codex and Claude use the latest canonical
target proposal rules, then check doctor and memory-report for any proposal
actions that still need review."

## Step 1: Refresh agent setup

```bash
mbrain setup-agent --apply
```

This refreshes managed Codex and Claude prompt rules so agents create canonical
target proposals for targetless candidates, wait for approval before binding,
and route missing-page stubs through patch-candidate review.

## Step 2: Verify installed readiness

```bash
mbrain doctor --agent --explain --json
```

Check that:

- the installed CLI version is `0.14.1`
- Codex and Claude MCP registration are ok or intentionally absent
- prompt rules match the packaged `docs/MBRAIN_AGENT_RULES.md`
- Claude stop and prompt hooks are ok when Claude Code is installed

## Step 3: Review canonical target proposal actions

```bash
mbrain memory-report
```

The report can now surface proposal lifecycle actions: create proposals for
targetless candidates, approve or reject proposed targets, complete approved
bindings, and review missing-page stub patch candidates before binding. No
schema migration or brain content rewrite is required.
