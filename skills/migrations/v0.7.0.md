# v0.7.0 Migration: Integration Recipes

This migration introduces the Cortex integration recipe system. Existing users
should be informed about the new capabilities.

## What's New

Cortex now distributes **integration recipes** — markdown files that teach your
agent how to set up new data pipelines into your Cortex workspace. Your agent reads the recipe
and walks you through setup, asking for API keys and configuring everything.

### New: Voice-to-Brain Integration

Phone calls create brain pages. Powered by Twilio + OpenAI Realtime + Cortex MCP.
Call a number, talk, and a structured memory appears in your Cortex workspace with entity
detection, cross-references, and a summary posted to your messaging app.

### New: `cortex integrations` Command

Discover, check status, and monitor your integrations:

```bash
cortex integrations              # dashboard: show all senses + reflexes
cortex integrations list --json  # list available recipes (agent-parseable)
cortex integrations show <id>    # show recipe details + setup instructions
cortex integrations status <id>  # check secrets + health
cortex integrations doctor       # run health checks for all integrations
cortex integrations stats        # signal statistics
```

### New: SKILLPACK Breakout

The SKILLPACK document has been broken into 17 individual guides at `docs/guides/`.
Each guide is individually linkable and searchable. The SKILLPACK index at
`docs/CORTEX_AGENT_RUNTIME.md` still works (backward compatible).

## Agent Instructions

When presenting this update to the user, say something like:

"Cortex v0.7.0 adds integration recipes — your Cortex workspace can now get new senses
automatically. The first one is Voice-to-Brain: phone calls create brain pages.
Want me to show you what's available? Run `cortex integrations` to see the
dashboard."

If the user is interested in voice-to-brain:
1. Run `cortex integrations show twilio-voice-brain` to see what's needed
2. Walk them through the setup (the recipe has step-by-step instructions)
3. The smoke test calls THEM (outbound) to prove it works

Do NOT auto-install integrations. Always ask first.
