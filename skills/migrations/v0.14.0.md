# v0.14.0 Migration: Refresh Agent Setup And Review Report Health

This release tightens the installed-agent surface and the memory review report.
Existing installs should refresh managed prompt and hook assets so Codex and
Claude Code use the packaged rules, Claude prompt/stop hooks, and the default
candidate-only maintenance schedule.

## Agent Instructions

When presenting this update, say something like:

"v0.14.0 makes the daily memory report more useful: connector rows now tell you
whether a source is stale, rate-limited, unauthorized, schema-broken, or ready
to retry. I should refresh setup-agent so Codex and Claude use the latest
managed prompt rules and hooks."

## Step 1: Refresh agent setup

```bash
mbrain setup-agent --apply
```

This refreshes the managed prompt rules for Codex and Claude Code, updates the
Claude hook scripts and registrations, and keeps the default candidate-only
maintenance schedule aligned with the installed package. Use `--preview` first
if the user wants to inspect changes before writing.

## Step 2: Verify installed readiness

```bash
mbrain doctor --agent --explain --json
```

Check that:

- the installed CLI version is `0.14.0`
- Codex and Claude MCP registration are ok or intentionally absent
- prompt rules match the packaged `docs/MBRAIN_AGENT_RULES.md`
- Claude stop and prompt hooks are ok when Claude Code is installed
- autopilot schedule status is healthy, intentionally skipped, or `enabled,
  last cycle not recorded` before the first scheduled 03:00 cycle has run

## Step 3: Review the memory report

```bash
mbrain memory-report
mbrain memory-report --save --report-dir <brain-dir>
```

Connector health rows now include staleness, retry posture, failure class, and a
plain next action. Treat `auth_expired` as a reauthentication task,
`rate_limited` as a wait/retry-window task, `schema` as a mapper inspection
task, and `network` or stale rows as retry candidates after checking service
health.

No schema migration or brain content rewrite is required.
