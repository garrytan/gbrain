# v0.10.0 Migration: Obsidian Graph Sync

This release adds deterministic Obsidian graph import. Existing users should run
the schema migration, backfill their vault links in dry-run mode, inspect the
report, then apply it.

## What Changed

### Typed link identity

Links are now unique by source page, target page, and link type. This lets an
`obsidian_link` edge coexist with a curated relationship edge between the same
two pages.

Run:

```bash
gbrain init
```

`gbrain init` is idempotent and applies schema migration v5.

### Obsidian link backfill

Preview the graph import first:

```bash
gbrain obsidian-link-sync --repo ~/brain --dry-run
```

Review:
- unresolved links
- ambiguous basename matches
- slug collisions
- links that would be added, updated, or removed

Then apply:

```bash
gbrain obsidian-link-sync --repo ~/brain
```

After this, regular `gbrain import` and `gbrain sync` runs keep the Obsidian
graph mirrored automatically.

## Agent Instructions

1. Run `gbrain init`.
2. Run `gbrain obsidian-link-sync --repo <brain-repo> --dry-run`.
3. If the report has slug collisions, fix the filenames first.
4. If unresolved or ambiguous links look intentional, explain them to the user.
5. Run `gbrain obsidian-link-sync --repo <brain-repo>`.
6. Run `gbrain health` and confirm orphan count drops when the vault had links.
