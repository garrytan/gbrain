# v0.13.0 Migration: Non-Blocking Claude Hooks

This release reworks the Claude Code hooks that `mbrain setup-agent` installs.
The old Stop hook blocked every eligible session end with
`{"decision":"block",...}`, which Claude Code displays as `Stop hook error` and
answers with a reflexive `MBRAIN-PASS: ...` reply. The new design follows the
documented non-blocking pattern:

- A new **UserPromptSubmit hook** (`prompt-mbrain-context.sh`) silently injects
  MBrain retrieval/writeback guidance as `additionalContext` on each prompt.
- The **Stop hook** is now silent by default (logs only). The legacy blocking
  memory gate is opt-in per session via `MBRAIN_STOP_HOOK_MODE=block`.
- `~/.claude/mbrain-skip-dirs` is no longer overwritten on re-runs, so user
  entries survive.

**Existing installs keep the OLD blocking Stop hook until the hooks are
reinstalled.** This migration's only required step is the reinstall below.

## Agent Instructions

When presenting this update, say something like:

"v0.13.0 replaces the noisy end-of-session MBrain check with silent per-prompt
retrieval guidance. Your sessions will no longer show 'Stop hook error' or
forced MBRAIN-PASS replies. I need to re-run setup-agent once to install the
new hooks."

## Step 1: Reinstall the Claude hooks

```bash
mbrain setup-agent --claude --skip-mcp --apply
```

This replaces `~/.claude/scripts/hooks/stop-mbrain-check.sh`, installs
`~/.claude/scripts/hooks/prompt-mbrain-context.sh`, refreshes the shared
relevance library, and registers the `prompt:mbrain-context` UserPromptSubmit
entry in `~/.claude/settings.json` (existing user settings are preserved; a
malformed settings.json is left untouched with a warning).

If MCP registration should also be (re)checked, drop `--skip-mcp`.

## Step 2: Verify

```bash
mbrain doctor --agent --explain
```

The readiness report now includes a `claude_prompt_hook` check. Both
`claude_stop_hook` and `claude_prompt_hook` should report ok.

## Step 3: Tell the user about the new controls

- `MBRAIN_STOP_HOOK_MODE=block claude` — restore the old blocking memory gate
  for one session (Claude Code will label it `Stop hook error`; that is UI
  wording, not a crash).
- `MBRAIN_PROMPT_HOOK=0 claude` — disable the per-prompt context note.
- `MBRAIN_STOP_HOOK=0 claude` — disable the stop-hook memory check entirely.
- `~/.claude/mbrain-skip-dirs` — absolute paths of directories where both hooks
  stay quiet.

No database changes, schema migrations, or brain content changes are involved.
