# MVP shipping sequence

## 0. Define the core data model first

Even before identity generation is good, make the product architecture identity-first.

### Core objects

| Object | Meaning |
|---|---|
| **Identity** | Real artist, virtual artist, influencer, character, founder, model |
| **Song** | Track, hook, stems, lyrics, genre, mood, BPM |
| **Scene** | Generic scene, LA location, performance setup, lifestyle moment |
| **Render** | The actual TikTok/Reel/Short output |
| **Memory** | What worked, what the artist liked, what performed |

The primitive is:

```text
Identity + Song + Scene = Render
```

Day one identity schema:

```text
Identity
- owner_user_id
- identity_type: real_person | virtual_character
- display_name
- visual_references[]
- style_card
- music_assets[]
- voice_profile: null initially
- consent_status
- allowed_output_types
- prohibited_output_types
- generated_assets[]
- performance_memory[]
```

This lets you start with generic videos while already laying the foundation for locked identities.
