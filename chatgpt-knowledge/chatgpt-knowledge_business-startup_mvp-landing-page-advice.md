## Answer

Mr. White,

You’re asking the *right* question. The tension you’re feeling (vision vs. MVP, investor vs. user) is exactly where great landing pages are made.

Below is a **clear decision**, then a **repeatable process**, then a **concrete page blueprint** you can execute immediately.

---

## The Core Decision (this resolves 80% of the confusion)

**Your landing page should sell the *outcome*, not the MVP mechanics — but it must only *promise* what the MVP can honestly deliver today.**

Put differently:

- **Investors** buy *why this becomes huge*
- **Users** buy *why this helps me right now*
- **Your landing page must do both without lying**

That means:
- **Lead with the end-state benefit** (AI-optimized fan conversion)
- **Demonstrate credibility using the MVP reality** (clean link-in-bio + audience capture + optimization)
- **Never list roadmap features as product features**
- **Never explain the MVP as “basic” or “early”**

The user doesn’t care that it’s an MVP. They care whether it solves *their problem better than Linktree*.

---

## What the Landing Page *Is* (and Is Not)

### It **IS**
- A *belief installer*: “This understands artists”
- A *positioning weapon* against Linktree
- A *trust signal* for YC-level readers
- A *filter* for the right early users

### It is **NOT**
- A feature spec
- A roadmap
- A technical explanation
- An investor pitch deck

---

## The Correct Mental Model

Think of the page as answering **one question per scroll**:

1. *Is this for me?*
2. *Is this different?*
3. *Does this actually work?*
4. *Can I trust these people?*
5. *What do I do next?*

If any scroll fails → bounce.

---

## The Process to Craft a World-Class Landing Page

### Step 1: Write the “One-Sentence Truth”
This is not marketing copy. This is positioning.

> **“An AI-powered link-in-bio that turns casual listeners into real fans.”**

Everything else must support that sentence.

If a section doesn’t reinforce it, delete it.

---

### Step 2: Define the Enemy (Explicitly)
You are not competing with “tools.”
You are competing with **choice overload**.

Your villain:
> “Ten identical links and no idea what fans actually do.”

This framing is critical. Without it, you’re just “another link page.”

---

### Step 3: Separate *Promise* from *Proof*
This is where most founders screw up.

- **Promise** = what the system ultimately optimizes for
- **Proof** = what exists *today* that makes the promise believable

You *can* say:
> “Automatically optimized for every fan.”

You *must* then show:
- Single primary CTA
- Subscribe-first flow
- Visitor-level analytics
- Continuous testing (even if simple)

This keeps you honest and credible.

---

### Step 4: Decide the One CTA (you already did)
You’re correct:
> **Request Early Access**

Do **not** add secondary CTAs.
Do **not** add “Learn more” buttons.
Do **not** let people wander.

YC *loves* restraint.

# Landing Page Design Blueprint (V1)

---

## GLOBAL DESIGN RULES (DO NOT BREAK)

- **One-column flow** (no zig-zag storytelling)
- **Max width:** ~1100–1200px content
- **Generous vertical spacing** (this signals confidence)
- **Neutral palette** (black / off-white / gray + 1 accent)
- **No gradients**
- **No decorative illustrations**
- **No stock photos**
- **No logos unless meaningful**
- **Screenshots > mockups > illustrations**
- **Text always wins over visuals**

---

## HERO SECTION

### Component Type
**Centered Hero + Right-Aligned Product Screenshot (Desktop)**  
**Stacked (Mobile)**

### Layout
- Left: Text (headline, subheadline, CTA)
- Right: **Single product screenshot** (not a collage)

### Why
- F-pattern scanning
- Immediate grounding in reality
- YC reads this as “real product”

### Visual Element
**Actual screenshot** of:
- Artist profile page
- Clean
- No annotations
- No arrows
- No captions

If screenshot isn’t polished yet:
→ Use a **cropped partial** showing the CTA + layout only

### CTA
- Primary button only
- Solid fill
- No secondary links

### Avoid
- Background patterns
- Video
- Animated transitions
- Rotating screenshots

---

## PROBLEM SECTION

### Component Type
**Text-First, Narrow Column**

### Layout
- Centered title
- Left-aligned body copy underneath
- Bullets stacked vertically

### Visuals
**None**

This section should feel almost boring.  
That’s a good thing.

### Why
- Forces reader to confront the pain
- Increases cognitive contrast with later sections
- YC reads clarity as confidence

---

## INSIGHT SECTION

### Component Type
**Statement Block + Subtle Divider**

### Layout
- Large statement headline
- Short paragraphs beneath
- Optional thin horizontal rule above or below

### Visuals
**None**

If you add anything:
- A **single subtle icon** (optional)
- Monochrome only

### Why
This is your “aha” moment.  
Visuals dilute it.

---

## WHAT YOU GET (FEATURES)

### Component Type
**Two-Column Feature Grid (Text-Heavy)**

### Layout
- 2 columns on desktop
- Single column on mobile
- Each feature = short title + 1–2 lines

### Visuals
**No screenshots per feature**

Optional:
- Tiny, consistent icons (outline, monochrome)
- Icons must not compete with text

### Why
- Prevents feature bloat
- Keeps focus on outcomes
- Avoids “SaaS landing page noise”

---

## DIFFERENTIATION SECTION

### Component Type
**Side-by-Side Comparison (Text Only)**

### Layout
- Two columns
- Clear headings
- Bullet lists

### Visuals
**None**

Optional:
- Very subtle background tint for contrast
- No borders
- No checkmarks (feels marketing-y)

### Why
This is where conversion happens logically, not emotionally.
