- **No user token needed** unless you want personal library data or playback authorization

So the rule is:

| Task | Requires Developer Token | Requires User Token |
|---|---|---|
| Search catalog artists | **Yes** | No |
| Display catalog metadata (artwork, albums, genres) | **Yes** | No |
| Playback | **Yes** | Yes |
| Access a user’s personal playlists or library | **Yes** | **Yes** |

---

# **Why You Need It**
Apple strictly gates their catalog with a signed JWT to:

- authenticate your app
- control rate limits
- protect catalog licensing

Even though you’re not touching private user data, Apple doesn’t allow anonymous hits to their catalog search endpoints.

No token = **requests are rejected**.

---

# **The ONLY case where you don’t need an Apple key**
If you **proxy search through a third-party API** (e.g., Napster, Deezer, Discogs, MusicBrainz), then:

- You can search artists and get metadata **without an Apple Music token**
- BUT: data will not be Apple Music–specific
- You can’t deep-link into Apple Music albums/songs unless you match IDs manually

For an Apple-centric onboarding UX, that’s a bad trade.

To onboard musicians quickly and look legit:

- Use **Apple Music Catalog API with a developer token**
- Search artists directly with Apple’s IDs
