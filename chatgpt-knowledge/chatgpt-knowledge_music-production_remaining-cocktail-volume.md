## Question

Can i have my next.js app pull info from a google sheet?

## Answer

Absolutely, Mr. White. Easiest options:

# 1) Fast & public (no auth): Publish as CSV/JSON
- In Google Sheets: File → Share → “Publish to the web” → select the sheet → CSV.
- Fetch from Next.js (server or client):

```ts
// app/api/sheet/route.ts (Next.js 13+)
import { NextResponse } from "next/server";

export async function GET() {
  const url = "https://docs.google.com/spreadsheets/d/<SHEET_ID>/export?format=csv&id=<SHEET_ID>&gid=<GID>";
  const csv = await fetch(url, { cache: "no-store" }).then(r => r.text());
  // tiny CSV → JSON
  const [header, ...rows] = csv.trim().split("\n").map(r => r.split(","));
  const data = rows.map(r => Object.fromEntries(r.map((v,i)=>[header[i], v])));
  return NextResponse.json(data);
}
```

Pros: dead simple.  
Cons: sheet is public (read-only).

# 2) Private & secure: Google Sheets API (service account)
**Setup (once):**
1) In Google Cloud → APIs & Services: enable **Google Sheets API**.
2) Create a **Service Account**; generate a JSON key.
3) Share the sheet with the service account email (Viewer or Editor).
4) In Next.js, add env vars: `GOOGLE_SERVICE_ACCOUNT_EMAIL`, `GOOGLE_SERVICE_ACCOUNT_KEY` (the **private key**, with `\n` newlines), `SHEET_ID`.

**Server-only code (Route Handler):**
```ts
// app/api/sheet/route.ts
import { NextResponse } from "next/server";
import { google } from "googleapis";

export async function GET() {
  const auth = new google.auth.JWT(
    process.env.GOOGLE_SERVICE_ACCOUNT_EMAIL!,
    undefined,
    process.env.GOOGLE_SERVICE_ACCOUNT_KEY!.replace(/\\n/g, "\n"),
    ["https://www.googleapis.com/auth/spreadsheets.readonly"]
  );
  const sheets = google.sheets({ version: "v4", auth });
  const res = await sheets.spreadsheets.values.get({
    spreadsheetId: process.env.SHEET_ID!,
    range: "Sheet1!A1:Z1000", // adjust
  });

  const values = res.data.values ?? [];
  if (!values.length) return NextResponse.json([], { status: 200 });

  const [header, ...rows] = values;
  const data = rows.map(r => Object.fromEntries(header.map((h,i)=>[h, r[i] ?? ""])));
  return NextResponse.json(data, { headers: { "cache-control": "no-store" }});
}
```

**Client usage:**
```ts
// any client/server component
async function getData() {
  const res = await fetch("/api/sheet", { cache: "no-store" });
  return res.json();
}
```

Pros: private, controlled access.  
Cons: small setup.

# Notes
- Keep credentials **server-side only** (Route Handlers / Server Actions). Never expose the key to the browser.
- If you just need read-only public data without auth, Option 1 is fine. For anything private or future-proof, use Option 2.
- For perfs, add caching (e.g., `revalidate` or an in-memory/edge cache) if your sheet doesn’t change often.
