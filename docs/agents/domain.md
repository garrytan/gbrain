# Domain Docs

This repo uses a single-context domain documentation layout.

Before using engineering skills that depend on domain language or architectural decisions, read:

- `CONTEXT.md` at the repo root, if present
- `docs/adr/`, if present
- the relevant current-state architecture docs under `docs/architecture/`

If these files do not exist, proceed silently. Do not create them unless the user explicitly asks or a domain-decision task requires it.
