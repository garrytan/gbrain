# Action Brain 0.2 QA Runbook (48h Observation)

This runbook validates the post-ship behavior for Action Brain 0.2
(`GIT-1057`, Child `GIT-1065`) over a continuous 48-hour production window.

## Pass/Fail Contract

Pass requires all conditions:
- Zero drafts were sent without explicit user approval.
- 100% of sent drafts have a matching `draft_approved` event before `draft_sent`.
- No evidence of automatic send behavior in logs or history.

Fail if any sent draft lacks a prior approval event, or if any automation path
can send a message without a human `approve` action.

## Preconditions

- `GIT-1061`, `GIT-1062`, `GIT-1063`, and `GIT-1064` are already landed.
- Gold-set gate is green on `master`.
- Deploy is live and stable.
- QA has shell access with `DATABASE_URL` for read-only verification.

## Observation Window

1. Pick an exact start timestamp in UTC (example: `2026-04-20T01:00:00Z`).
2. Run all baseline queries at start (`T+0`).
3. Repeat the same queries at `T+24h`.
4. Run the final snapshot at `T+48h`.
5. Use one fixed window: `[window_start, window_start + interval '48 hours')`.

## Evidence Queries (Postgres)

Replace `:window_start` with your ISO timestamp.

```sql
-- 1) Draft lifecycle counts during the window.
SELECT status, COUNT(*) AS count
FROM action_drafts
WHERE generated_at >= :window_start::timestamptz
  AND generated_at < (:window_start::timestamptz + interval '48 hours')
GROUP BY status
ORDER BY status;

-- 2) History event counts during the window.
SELECT event_type, COUNT(*) AS count
FROM action_history
WHERE timestamp >= :window_start::timestamptz
  AND timestamp < (:window_start::timestamptz + interval '48 hours')
  AND event_type IN (
    'draft_created',
    'draft_approved',
    'draft_rejected',
    'draft_sent',
    'draft_send_failed'
  )
GROUP BY event_type
ORDER BY event_type;

-- 3) Sent drafts that do NOT have a matching approval within 1s (must be 0).
WITH sent AS (
  SELECT
    item_id,
    metadata->>'draft_id' AS draft_id,
    timestamp AS sent_at
  FROM action_history
  WHERE event_type = 'draft_sent'
    AND timestamp >= :window_start::timestamptz
    AND timestamp < (:window_start::timestamptz + interval '48 hours')
),
approved AS (
  SELECT
    item_id,
    metadata->>'draft_id' AS draft_id,
    timestamp AS approved_at
  FROM action_history
  WHERE event_type = 'draft_approved'
)
SELECT COUNT(*) AS sent_without_approval
FROM sent s
WHERE NOT EXISTS (
  SELECT 1
  FROM approved a
  WHERE a.item_id = s.item_id
    AND a.draft_id = s.draft_id
    AND a.approved_at <= s.sent_at
    AND a.approved_at >= s.sent_at - interval '1 second'
);

-- 4) Approval coverage ratio for sent drafts (must be 1.0).
WITH sent AS (
  SELECT
    item_id,
    metadata->>'draft_id' AS draft_id,
    timestamp AS sent_at
  FROM action_history
  WHERE event_type = 'draft_sent'
    AND timestamp >= :window_start::timestamptz
    AND timestamp < (:window_start::timestamptz + interval '48 hours')
),
approved AS (
  SELECT
    item_id,
    metadata->>'draft_id' AS draft_id,
    timestamp AS approved_at
  FROM action_history
  WHERE event_type = 'draft_approved'
),
approved_sent AS (
  SELECT s.item_id, s.draft_id, s.sent_at
  FROM sent s
  WHERE EXISTS (
    SELECT 1
    FROM approved a
    WHERE a.item_id = s.item_id
      AND a.draft_id = s.draft_id
      AND a.approved_at <= s.sent_at
      AND a.approved_at >= s.sent_at - interval '1 second'
  )
)
SELECT
  (SELECT COUNT(*) FROM approved_sent)::float
  / NULLIF((SELECT COUNT(*) FROM sent), 0)::float AS approval_coverage_ratio;
```

## CLI Cross-Checks

Use CLI output to corroborate SQL snapshots:

```bash
gbrain action draft list
gbrain action draft list --status send_failed
gbrain action brief
```

Verify:
- All sent drafts correspond to explicit `gbrain action draft approve <draft_id>` usage.
- Any send failure is visible and remains non-sent until explicit re-approval flow.

## Reporting Template (Issue Comment)

Post one comment at `T+48h` with:

- Window start/end (UTC).
- Snapshot at `T+0`, `T+24h`, `T+48h`.
- `sent_without_approval` result.
- Approval coverage ratio.
- `send_failed` count and any remediation.
- Verdict: `PASS` or `FAIL`.

Example verdict line:
- `PASS: 0 sent_without_approval; approval_coverage_ratio=1.0; no auto-send evidence in 48h window.`
