## Architecture deltas

All work lives inside `src/action-brain/` and `test/action-brain/`. No cross-cutting touchpoints outside Action Brain except:

- **GIT-174** — counters surface through the existing run summary pathway used by `action_ingest` / `action_brief`. Add fields to the existing return shape; do NOT introduce a new metrics framework.
- **GIT-175** — gold-set fixture lives at `test/action-brain/fixtures/gold-set.jsonl`. CI gate added to `bun test` (unit lane), not the E2E pgvector container.
- **GIT-177** — calls `engine.searchKeyword` / page resolver from GBrain core. Use existing operations contract, do not bypass.

### Data flow (commitment extraction post-0.1c)

```text
WhatsApp message
  → action_ingest (trust boundary: remote=true)
  → extractor.ts
      ├─ extractCommitments() ← retry/backoff (GIT-172)
      ├─ counters++ (GIT-174)
      └─ low_confidence drop → action_history.dropped (GIT-173)
  → action-engine.upsert() ← unchanged
  → brief.ts
      └─ entity-link source_contact (GIT-177)
```

### Trust boundaries (preserve from v0.13.0)

- Owner sanitizer in `extractor.ts` stays. Do not regress.
- LLM prompt interpolation discipline (XML delimiter defense) stays. Audit additions in GIT-173 must use the same delimiter pattern when echoing source content.
- Retry path (GIT-172) MUST be idempotent — same source_id ⇒ same upsert result. No duplicate commitments on retry.

- **Entity linking false positives** — GIT-177 must degrade cleanly. A confidently wrong link is worse than no link.
