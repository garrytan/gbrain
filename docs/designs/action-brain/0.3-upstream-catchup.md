## Why this order is locked

1. `#210` introduces the Knowledge Runtime substrate (Resolver SDK, BrainWriter path, integrity command, Budget/Completeness plumbing) that downstream jobs and graph-edge extraction rely on.
2. `#217` extends the minions job system with shell job handlers and abort-path hardening; it assumes the refreshed runtime/queue behavior from `#210`.
3. `#231` upgrades extraction to frontmatter relationship indexing + typed graph edges and schema/migration changes; it must run last to minimize conflict surface while adopting prior runtime changes first.

## Merge mechanics (locked)

For each child:
1. Branch from latest `master` using `impl/git-1066-child-<n>-<slug>` naming.
2. Absorb upstream commit set by cherry-picking upstream history in-order for that PR scope (preferred), or equivalent patch import if cherry-pick is blocked by mechanical conflicts.
3. Resolve conflicts with minimal edits, preserving upstream semantics.
4. Run full local tests (`bun test`) before `/ship`.
5. Ship via `/ship` only; do not hand-roll PR flow.
